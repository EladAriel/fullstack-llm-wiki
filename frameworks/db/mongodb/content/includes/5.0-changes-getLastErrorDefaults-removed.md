Starting in MongoDB 5.0, you cannot specify a default write concern with :rsconf:`settings.getLastErrorDefaults` other than the default of `{ w: 1, wtimeout: 0 }` . Instead, use the :dbcommand:`setDefaultRWConcern` command to set the default read or write concern configuration for a replica set or sharded cluster.
