## Prerequisites

### Feature Compatibility Version

The |oldversion| instance must have `featureCompatibilityVersion` set to |oldfcv|. To check `featureCompatibilityVersion`:

```javascript
db.adminCommand( { getParameter: 1, featureCompatibilityVersion: 1 } )
```

The operation returns a document with the current feature compatibility version:

```javascript
{ featureCompatibilityVersion : { version : "6.0" }, ok: 1}
```

To set or update `featureCompatibilityVersion` to `6.0`, run the following command:

```javascript
db.adminCommand( { setFeatureCompatibilityVersion: "6.0" } )
```

For more information, see :dbcommand:`setFeatureCompatibilityVersion`.

### Consider Converting to Replica Set

Prior to upgrading, consider `converting your standalone deployment to a replica set <server-replica-set-deploy-convert>`. Replica sets are the recommended deployment configuration for MongoDB.
