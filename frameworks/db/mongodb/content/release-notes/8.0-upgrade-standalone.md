## Prerequisites

### Feature Compatibility Version

The |oldversion| instance must have `featureCompatibilityVersion` set to |oldfcv|. To check `featureCompatibilityVersion`, run the following command:

```javascript
db.adminCommand( { getParameter: 1, featureCompatibilityVersion: 1 } )
```

The operation should return a result that includes `"featureCompatibilityVersion" : { "version" : "7.0" }`.

To set or update `featureCompatibilityVersion`, run the following command:

```javascript
db.adminCommand( { setFeatureCompatibilityVersion: "7.0", confirm: true } )
```

For more information, see :method:`db.adminCommand()` and :dbcommand:`setFeatureCompatibilityVersion`.

### Consider Converting to Replica Set

Prior to upgrading, consider `converting your standalone deployment to a replica set <server-replica-set-deploy-convert>`. Replica sets are the recommended deployment configuration for MongoDB.
