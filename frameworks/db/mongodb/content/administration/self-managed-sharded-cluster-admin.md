# Self-Managed Sharded Cluster Administration

These documents provide instructions on how to administer a self-managed sharded cluster.

`replace-self-managed-config-server` Replace a config server in a replica set.

`restart-sharded-cluster` Stop and restart a sharded cluster.

`migrate-cluster-to-different-hardware` Migrate a sharded cluster to a different hardware system, for example, when moving a pre-production environment to production.

`back-up-sharded-cluster-metadata` Create a backup of a sharded cluster's metadata while keeping the cluster operational.

`convert-sharded-cluster-to-replica-set` Convert a sharded cluster into a single replica set.

`manual-convert-replica-set-to-sharded-cluster` Deploy a sharded cluster for an existing replica set.
